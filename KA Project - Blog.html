14th Post:

Hello there! It's been more than 3 months since I've written on this blog, I really don't know why. Maybe it's because i was just being lazy. Anyway, what I want to talk about is something amazing: Arch Linux! Arch Linux is a Linux distro "focused on keeping it simple." The best part about this is that, unlike Ubuntu, which was the first distro I used, it isn't bloated. It's a custom install that only has what you put their, nothing more and nothing less. Now don't get me wrong, Windows is 10 times more bloated than even Ubuntu, but Arch Linux is the next. Obviously I could have gone with an even more minimalist or customizable distro, like Gentoo or Artix Linux, but I felt that for now Arch Linux will be enough to satisfy me and be efficient enough.

You start off with an iso that, when booted, gives a blank terminal screen. You then run various commands to install your system. Obviously, I won't go through the installation process here, but the point is that you get what you want. After I finished installing, I only had a total of 450 packages installed and 5.1 gigabytes of disk spaces used. That's because, in reality, it's all I need in my Linux system, so it's all I did during my Arch installation. By comparison, Ubuntu comes with 2000 packages, many of which I don't need. You might ask, why does Ubuntu bloat up their operating system when they don't have to? Well, that's because Ubuntu is focused on being a user-f4riendly distro, whereas Arch will always be a user-centric distro. After using Linux for a few months and even making it my daily driver over Windows 10 (which I now hate), I've come to the point where I know enough about Linux and am smart enough to use something that is more focused on speed, efficiency, and user-centricity rather than something (like Ubuntu or even Fedora) that does everything for you and makes it easier. And yes, while using Arch Linux did make things harder for me at the beginning, I always found a solution, and also found many advantages to using Arch Linux. In my journey with Arch Linux (it's been like 1 month), I've tried using minimalist software. With some experience, I can easily say that using software like Neovim over Visual Studio Code, and Zathura over Adobe Acrobat is just so much faster, easier, and just plain better in general. Configuring apps through a config file in your .config folder in your home directory is much faster and more hackable than doing it with GUI settings. Working through the terminal is an added plus. Working in a terminal is so much faster than using anything else. If I could browse the web through the terminal effectively, I would do it (I can't, sadly). The terminal is a tool that is overlooked by a lot of people. This is because people think that its meant solely to interact with the deep ends of a system and people think that they could potentially harm their comptuer using it. However, the terminal is a very friendly and effective tool to speed up your workflow that makes you think what you would be without it once you get good at using it. No matter what shell you use, bash or zsh or any other shell, the terminal is amazing. Say you want to output the text of a file but don't want to open it up in a text editor. All you need to do is type in cat path_to_file and its done. If you only want to output lines of that file that have specific key words, append grep at the end of it: cat path_to_file | grep "keyword". This makes life so much easier when dealing with configurations and you need specific information about a window, package, font, or anything, really. Installing something like tmux or byobu allows you to have a window manager inside of your terminal, so you can manage different terminal windows inside the terminal itself, which is amazing for development. I haven't covered everything here, but there is so much to the terminal that I and a lot of other people appreciate.

This brings me to my next point. This may sound stupid, but using Arch Linux has increased my IQ, and particularly my problem solving skills. Before, I would need endless Google searches simply to figure out how to install an app on Ubuntu, or fix a small problem. Now, I've become accustomed to fixing my own problems and am capable of finding the solution without needing help. I have a better understanding of terminal commands, and can use logic to find a solution to a problem rather than simply copy pasting commands from some random website into a terminal. Best thing is, this is the longest time I've gone without breaking my linux distro while still making huge changes to my distro as well. So that's all I have for now, and I'll end off with one thing: GNU/Linux is the best operating system hands down, and macOS and Windows don't even compare.
